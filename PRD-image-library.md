# FLOKI-IMAGE-LIBRARY.md

Deployment brief for the Image Library admin tool at `/admin/images`.

**What this is:** A single-purpose admin page where Jesper uploads, describes, and tags every image on the site, plus a `findImages()` helper you (Floki) call before assigning any image in any post or tool. Solves the "Floki picked timber framing when asked for plywood" problem.

**What it isn't:** A public gallery. A CMS replacement. A generic media library. Admin-only, admin-tagged, admin-curated.

**Architecture at a glance:**

- Images stored in Vercel Blob (new uploads) plus the existing `public/images/` (legacy, indexed in place — don't move them).
- Metadata in Postgres in a new `images` table. Both sources queryable the same way.
- Admin UI at `/admin/images`, gated by `ADMIN_EMAIL` (same pattern as `/admin/title-lab`).
- AI auto-description and auto-tagging on upload via Claude Sonnet 4 through OpenRouter.
- Controlled vocabulary in `src/data/image-tag-vocabulary.ts`, plus freeform custom tags per image.
- A `findImages()` TypeScript helper + hard rule in `JESPER-MAKES-DEV-HANDOFF.md` that you (Floki) must consult before assigning image paths.

**One-line test it's working:** you open `/admin/images`, see all 97 existing images in a grid, each with an AI-generated description and tags, and when Jesper next asks for a plywood image you call `findImages({ material: "plywood" })` and get real matches instead of guessing from filenames.

  id: uuid("id").primaryKey().defaultRandom(),

  customTags: text("custom_tags").array().notNull().default(sql`ARRAY[]::text[]`),

  tagsAutoGenerated: boolean("tags_auto_generated").notNull().default(false),

  updatedAt: timestamp("updated_at", { withTimezone: true }).notNull().defaultNow(),
  uploadedByUserId: uuid("uploaded_by_user_id").references(() => users.id, { onDelete: "set null" }),
});

export type NewImage = typeof images.$inferInsert;
```

Make sure `sql` is imported from `drizzle-orm` at the top of the file if it isn't already:
```typescript
import { sql } from "drizzle-orm";
```

Run the migration:
```bash
npx drizzle-kit generate
POSTGRES_URL=$(grep '^POSTGRES_URL=' .env.local | head -1 | cut -d= -f2- | tr -d '"')
export POSTGRES_URL
npx drizzle-kit push
```

**Column notes:**
- `source` is either `"blob"` (Vercel Blob) or `"public"` (existing `/public/images/...`). Never changes after insert.
- `url` for `blob` source is the full `https://...blob.vercel-storage.com/...` URL. For `public` source it's the site-relative path (e.g. `/images/blog/plywood-closeup.jpg`).
- `pathname` is the cleaned-up path without the domain, used for display and uniqueness checks.
- `material`, `shotType`, `setting` are single-valued from the controlled vocabulary (or null). `subjects`, `who`, `customTags` are arrays.
- `reviewed` flips to `true` the first time Jesper saves the image from the admin UI. Until then, the UI shows the amber "untagged / unreviewed" dot even if AI populated fields.
- `hidden` is soft delete. We never hard-delete images — too easy to break live posts.

## 4. The `findImages()` helper (this is the important one)

Create `src/lib/image-library.ts`. This is the canonical query interface and the thing you (Floki) are required to use before picking any image.

```typescript
import { db } from "@/lib/db";
import { images, type Image } from "@/lib/db/schema";
import { and, or, eq, ilike, sql, desc, inArray } from "drizzle-orm";

export interface FindImagesOptions {
  material?: string;
  subjects?: string[];
  shotType?: string;
  who?: string[];
  setting?: string;
  customTags?: string[];
  query?: string;
  limit?: number;
  includeHidden?: boolean;
}

export interface ImageMatch extends Image {
  matchScore: number;
}

/**
 * Find images from the library. ALWAYS call this before assigning any
 * /images/* path in blog posts, tools, product pages, or any other file.
 *
 * Do NOT pick images by listing public/images/. Do NOT guess from filename.
 * If nothing matches well, stop and ask Jesper to upload — don't fall back
 * to a similar-looking file.
 *
 * @example
 * const matches = await findImages({ material: "plywood", shotType: "detail" });
 * // matches[0].url is the URL to use in heroImage/featuredImage
 *
 * @example
 * const matches = await findImages({ query: "workshop overview", limit: 5 });
 */
export async function findImages(options: FindImagesOptions = {}): Promise<ImageMatch[]> {
  const {
    material,
    subjects,
    shotType,
    who,
    setting,
    customTags,
    query,
    limit = 20,
    includeHidden = false,
  } = options;

  const conditions = [];

  if (!includeHidden) {
    conditions.push(eq(images.hidden, false));
  }

  if (material) {
    conditions.push(eq(images.material, material));
  }
  if (shotType) {
    conditions.push(eq(images.shotType, shotType));
  }
  if (setting) {
    conditions.push(eq(images.setting, setting));
  }

  if (subjects && subjects.length > 0) {
    conditions.push(sql`${images.subjects} && ${subjects}`);
  }
  if (who && who.length > 0) {
    conditions.push(sql`${images.who} && ${who}`);
  }
  if (customTags && customTags.length > 0) {
    conditions.push(sql`${images.customTags} && ${customTags}`);
  }

  if (query && query.trim()) {
    const q = `%${query.trim()}%`;
    conditions.push(
      or(
        ilike(images.description, q),
        ilike(images.filename, q),
        sql`EXISTS (SELECT 1 FROM unnest(${images.customTags}) AS t WHERE t ILIKE ${q})`,
        sql`EXISTS (SELECT 1 FROM unnest(${images.subjects}) AS s WHERE s ILIKE ${q})`
      )
    );
  }

  const rows = await db
    .select()
    .from(images)
    .where(conditions.length ? and(...conditions) : undefined)
    .orderBy(desc(images.reviewed), desc(images.updatedAt))
    .limit(limit);

  return rows.map((row) => ({
    ...row,
    matchScore: scoreMatch(row, options),
  })).sort((a, b) => b.matchScore - a.matchScore);
}

function scoreMatch(image: Image, options: FindImagesOptions): number {
  let score = 0;
  if (options.material && image.material === options.material) score += 5;
  if (options.shotType && image.shotType === options.shotType) score += 3;
  if (options.setting && image.setting === options.setting) score += 2;
  if (options.subjects) {
    const overlap = options.subjects.filter((s) => image.subjects.includes(s)).length;
    score += overlap * 3;
  }
  if (options.who) {
    const overlap = options.who.filter((w) => image.who.includes(w)).length;
    score += overlap * 2;
  }
  if (options.customTags) {
    const overlap = options.customTags.filter((t) => image.customTags.includes(t)).length;
    score += overlap * 2;
  }
  if (image.reviewed) score += 1;
  return score;
}

export async function getImageById(id: string) {
  const rows = await db.select().from(images).where(eq(images.id, id)).limit(1);
  return rows[0] ?? null;
}

export async function getImagesByIds(ids: string[]) {
  if (!ids.length) return [];
  return db.select().from(images).where(inArray(images.id, ids));
}
```

## 5. The AI description/tagging helper

Create `src/lib/image-ai.ts`. Called on upload and on "regenerate" from the admin UI.

```typescript
import { IMAGE_TAG_VOCABULARY } from "@/data/image-tag-vocabulary";

const MODEL = "anthropic/claude-sonnet-4";
const OPENROUTER_URL = "https://openrouter.ai/api/v1/chat/completions";

export interface AutoTagResult {
  description: string;
  material: string | null;
  subjects: string[];
  shotType: string | null;
  who: string[];
  setting: string | null;
}

export async function autoTagImage(imageUrl: string): Promise<AutoTagResult> {
  const vocab = IMAGE_TAG_VOCABULARY;

  const systemPrompt = `You are an image tagger for jespermakes.com, a Danish story-led woodworking YouTube channel.
You look at one image and return ONE JSON object describing it for the image library.

Return ONLY valid JSON, no markdown, no preamble, matching this shape:
{
  "description": "<one concise sentence, concrete, no fluff, no marketing language>",
  "material": <one of: ${vocab.material.values.join(", ")}, or null if no dominant wood>,
  "subjects": [<any of: ${vocab.subjects.values.join(", ")}>],
  "shotType": <one of: ${vocab.shotType.values.join(", ")}, or null>,
  "who": [<any of: ${vocab.who.values.join(", ")}, empty array if no person visible>],
  "setting": <one of: ${vocab.setting.values.join(", ")}, or null>
}

Rules:
- description: ONE sentence, max 20 words. Describe what's actually visible. No "beautiful", "stunning", "showcasing" or marketing words.
- material: the dominant wood type visible, or null if the image isn't wood-focused (e.g. portrait, landscape).
- subjects: every relevant subject from the list. Multiple OK. Empty array if none fit.
- shotType: pick the single best fit. Hero = wide/final-piece showpiece. Detail = close-up. Process = action mid-build. B-roll = ambient/contextual. Portrait = person-focused.
- who: only include if a person is clearly visible. "jesper" if it's clearly the channel host (bearded guy, often in wool/denim). "wife" if his partner is visible. "hands-only" if only hands are shown.
- setting: where was this shot? Null if unclear.

Never guess wildly. If unsure, use null or an empty array.`;

  const body = {
    model: MODEL,
    max_tokens: 600,
    messages: [
      { role: "system", content: systemPrompt },
      {
        role: "user",
        content: [
          { type: "image_url", image_url: { url: imageUrl } },
          { type: "text", text: "Tag this image. Return only the JSON object." },
        ],
      },
    ],
  };

  const response = await fetch(OPENROUTER_URL, {
    method: "POST",
    headers: {
      "Content-Type": "application/json",
      Authorization: `Bearer ${process.env.OPENROUTER_API_KEY}`,
      "HTTP-Referer": "https://jespermakes.com",
      "X-Title": "Jesper Makes Image Library",
    },
    body: JSON.stringify(body),
  });

  if (!response.ok) {
    const text = await response.text();
    throw new Error(`OpenRouter error ${response.status}: ${text.slice(0, 300)}`);
  }

  const data = await response.json();
  const raw = data.choices?.[0]?.message?.content ?? "";
  const parsed = extractJson(raw);
  return sanitize(parsed);
}

function extractJson(text: string): unknown {
  const clean = text.replace(/```json\s?|```/g, "").trim();
  const match = clean.match(/\{[\s\S]*\}/);
  if (!match) throw new Error("No JSON object found in AI response");
  return JSON.parse(match[0]);
}

function sanitize(raw: unknown): AutoTagResult {
  const obj = (raw ?? {}) as Record<string, unknown>;
  const vocab = IMAGE_TAG_VOCABULARY;

  const oneOf = (field: keyof typeof vocab, value: unknown): string | null => {
    if (typeof value !== "string") return null;
    const allowed = vocab[field].values as readonly string[];
    return allowed.includes(value) ? value : null;
  };

  const manyOf = (field: keyof typeof vocab, value: unknown): string[] => {
    if (!Array.isArray(value)) return [];
    const allowed = vocab[field].values as readonly string[];
    return value.filter((v): v is string => typeof v === "string" && allowed.includes(v));
  };

  return {
    description: typeof obj.description === "string" ? obj.description.slice(0, 300).trim() : "",
    material: oneOf("material", obj.material),
    subjects: manyOf("subjects", obj.subjects),
    shotType: oneOf("shotType", obj.shotType),
    who: manyOf("who", obj.who),
    setting: oneOf("setting", obj.setting),
  };
}
```

## 6. API routes

### 6.1 `src/app/api/admin/images/route.ts` — list + upload

```typescript
import { NextRequest, NextResponse } from "next/server";
import { put } from "@vercel/blob";
import imageSize from "image-size";
import { auth } from "@/lib/auth";
import { db } from "@/lib/db";
import { images } from "@/lib/db/schema";
import { eq, desc, sql, and, ilike, or } from "drizzle-orm";
import { autoTagImage } from "@/lib/image-ai";

export const runtime = "nodejs";
export const maxDuration = 60;

async function requireAdmin() {
  const session = await auth();
  if (!session?.user?.email || session.user.email !== process.env.ADMIN_EMAIL) {
    return null;
  }
  return session;
}

export async function GET(request: NextRequest) {
  const session = await requireAdmin();
  if (!session) return NextResponse.json({ error: "Unauthorized" }, { status: 401 });

  const { searchParams } = new URL(request.url);
  const query = searchParams.get("q")?.trim() || "";
  const untaggedOnly = searchParams.get("untagged") === "true";
  const material = searchParams.get("material") || "";
  const shotType = searchParams.get("shotType") || "";
  const tag = searchParams.get("tag") || "";

  const conditions = [eq(images.hidden, false)];

  if (untaggedOnly) {
    conditions.push(eq(images.reviewed, false));
  }
  if (material) conditions.push(eq(images.material, material));
  if (shotType) conditions.push(eq(images.shotType, shotType));
  if (tag) {
    conditions.push(
      sql`(${tag} = ANY(${images.subjects}) OR ${tag} = ANY(${images.customTags}) OR ${tag} = ANY(${images.who}))`
    );
  }
  if (query) {
    const q = `%${query}%`;
    conditions.push(
      or(
        ilike(images.description, q),
        ilike(images.filename, q),
        sql`EXISTS (SELECT 1 FROM unnest(${images.customTags}) AS t WHERE t ILIKE ${q})`,
      )!
    );
  }

  const rows = await db
    .select()
    .from(images)
    .where(and(...conditions))
    .orderBy(desc(images.reviewed), desc(images.updatedAt))
    .limit(500);

  const stats = await db
    .select({
      total: sql<number>`count(*)::int`,
      untagged: sql<number>`count(*) filter (where not ${images.reviewed})::int`,
    })
    .from(images)
    .where(eq(images.hidden, false));

  return NextResponse.json({
    images: rows,
    stats: stats[0] ?? { total: 0, untagged: 0 },
  });
}

export async function POST(request: NextRequest) {
  const session = await requireAdmin();
  if (!session) return NextResponse.json({ error: "Unauthorized" }, { status: 401 });

  const formData = await request.formData();
  const file = formData.get("file");

  if (!(file instanceof File)) {
    return NextResponse.json({ error: "No file provided" }, { status: 400 });
  }

  const maxBytes = 20 * 1024 * 1024;
  if (file.size > maxBytes) {
    return NextResponse.json({ error: "File too large (max 20MB)" }, { status: 413 });
  }

  const allowedTypes = ["image/jpeg", "image/png", "image/webp", "image/avif"];
  if (!allowedTypes.includes(file.type)) {
    return NextResponse.json({ error: "Unsupported file type" }, { status: 415 });
  }

  const arrayBuffer = await file.arrayBuffer();
  const buffer = Buffer.from(arrayBuffer);

  let width: number | null = null;
  let height: number | null = null;
  try {
    const size = imageSize(buffer);
    width = size.width ?? null;
    height = size.height ?? null;
  } catch {
  }

  const safeName = file.name.replace(/[^a-zA-Z0-9._-]/g, "-").toLowerCase();
  const blob = await put(`uploads/${safeName}`, buffer, {
    access: "public",
    addRandomSuffix: true,
    contentType: file.type,
  });

  let aiResult = null;
  try {
    aiResult = await autoTagImage(blob.url);
  } catch (err) {
    console.error("autoTagImage failed", err);
  }

  const [inserted] = await db
    .insert(images)
    .values({
      source: "blob",
      url: blob.url,
      pathname: blob.pathname,
      filename: file.name,
      width,
      height,
      sizeBytes: file.size,
      mimeType: file.type,
      description: aiResult?.description ?? null,
      material: aiResult?.material ?? null,
      subjects: aiResult?.subjects ?? [],
      shotType: aiResult?.shotType ?? null,
      who: aiResult?.who ?? [],
      setting: aiResult?.setting ?? null,
      descriptionAutoGenerated: !!aiResult?.description,
      tagsAutoGenerated: !!aiResult,
      reviewed: false,
      uploadedByUserId: session.user.id,
    })
    .returning();

  return NextResponse.json({ image: inserted });
}
```

### 6.2 `src/app/api/admin/images/[id]/route.ts` — get, update, hide

```typescript
import { NextRequest, NextResponse } from "next/server";
import { auth } from "@/lib/auth";
import { db } from "@/lib/db";
import { images } from "@/lib/db/schema";
import { eq } from "drizzle-orm";
import { IMAGE_TAG_VOCABULARY, isValidTagValue } from "@/data/image-tag-vocabulary";

export const runtime = "nodejs";

async function requireAdmin() {
  const session = await auth();
  if (!session?.user?.email || session.user.email !== process.env.ADMIN_EMAIL) return null;
  return session;
}

export async function GET(_req: NextRequest, { params }: { params: { id: string } }) {
  const session = await requireAdmin();
  if (!session) return NextResponse.json({ error: "Unauthorized" }, { status: 401 });

  const rows = await db.select().from(images).where(eq(images.id, params.id)).limit(1);
  if (!rows.length) return NextResponse.json({ error: "Not found" }, { status: 404 });
  return NextResponse.json({ image: rows[0] });
}

export async function PATCH(request: NextRequest, { params }: { params: { id: string } }) {
  const session = await requireAdmin();
  if (!session) return NextResponse.json({ error: "Unauthorized" }, { status: 401 });

  const body = await request.json();

  const updates: Record<string, unknown> = { updatedAt: new Date(), reviewed: true };

  if (typeof body.description === "string") {
    updates.description = body.description.slice(0, 500);
    updates.descriptionAutoGenerated = false;
  }
  if ("material" in body) {
    updates.material =
      body.material === null || body.material === ""
        ? null
        : isValidTagValue("material", body.material)
        ? body.material
        : null;
  }
  if ("shotType" in body) {
    updates.shotType =
      body.shotType === null || body.shotType === ""
        ? null
        : isValidTagValue("shotType", body.shotType)
        ? body.shotType
        : null;
  }
  if ("setting" in body) {
    updates.setting =
      body.setting === null || body.setting === ""
        ? null
        : isValidTagValue("setting", body.setting)
        ? body.setting
        : null;
  }
  if (Array.isArray(body.subjects)) {
    updates.subjects = body.subjects.filter((s: unknown): s is string =>
      typeof s === "string" && isValidTagValue("subjects", s)
    );
  }
  if (Array.isArray(body.who)) {
    updates.who = body.who.filter((s: unknown): s is string =>
      typeof s === "string" && isValidTagValue("who", s)
    );
  }
  if (Array.isArray(body.customTags)) {
    updates.customTags = body.customTags
      .filter((s: unknown): s is string => typeof s === "string")
      .map((s) => s.trim().toLowerCase().slice(0, 40))
      .filter((s) => s.length > 0 && s.length < 40)
      .slice(0, 20);
  }
  if (typeof body.tagsAutoGenerated === "boolean") {
    updates.tagsAutoGenerated = body.tagsAutoGenerated;
  }

  const [updated] = await db
    .update(images)
    .set(updates)
    .where(eq(images.id, params.id))
    .returning();

  if (!updated) return NextResponse.json({ error: "Not found" }, { status: 404 });
  return NextResponse.json({ image: updated });
}

export async function DELETE(_req: NextRequest, { params }: { params: { id: string } }) {
  const session = await requireAdmin();
  if (!session) return NextResponse.json({ error: "Unauthorized" }, { status: 401 });

  const [updated] = await db
    .update(images)
    .set({ hidden: true, updatedAt: new Date() })
    .where(eq(images.id, params.id))
    .returning();

  if (!updated) return NextResponse.json({ error: "Not found" }, { status: 404 });
  return NextResponse.json({ ok: true });
}

### 6.3 `src/app/api/admin/images/[id]/regenerate/route.ts` — re-run AI on one image

```typescript
import { NextRequest, NextResponse } from "next/server";
import { auth } from "@/lib/auth";
import { db } from "@/lib/db";
import { images } from "@/lib/db/schema";
import { eq } from "drizzle-orm";
import { autoTagImage } from "@/lib/image-ai";

export const runtime = "nodejs";
export const maxDuration = 60;

export async function POST(_req: NextRequest, { params }: { params: { id: string } }) {
  const session = await auth();
  if (!session?.user?.email || session.user.email !== process.env.ADMIN_EMAIL) {
    return NextResponse.json({ error: "Unauthorized" }, { status: 401 });
  }

  const rows = await db.select().from(images).where(eq(images.id, params.id)).limit(1);
  const image = rows[0];
  if (!image) return NextResponse.json({ error: "Not found" }, { status: 404 });

  const fullUrl = image.source === "public"
    ? `https://jespermakes.com${image.url}`
    : image.url;

  const result = await autoTagImage(fullUrl);

  const [updated] = await db
    .update(images)
    .set({
      description: result.description || image.description,
      material: result.material,
      subjects: result.subjects,
      shotType: result.shotType,
      who: result.who,
      setting: result.setting,
      descriptionAutoGenerated: true,
      tagsAutoGenerated: true,
      updatedAt: new Date(),
    })
    .where(eq(images.id, params.id))
    .returning();

  return NextResponse.json({ image: updated, suggested: result });
}

```typescript
import "dotenv/config";
import { readdirSync, statSync, readFileSync } from "fs";
import { join, relative, extname } from "path";
import imageSize from "image-size";
import { db } from "../lib/db";
import { images } from "../lib/db/schema";

const PUBLIC_IMAGES_DIR = join(process.cwd(), "public", "images");
const SUPPORTED_EXT = new Set([".jpg", ".jpeg", ".png", ".webp", ".avif"]);
const BASE_URL = "https://jespermakes.com";

function walk(dir: string, acc: string[] = []): string[] {
  for (const entry of readdirSync(dir)) {
    const p = join(dir, entry);
    const s = statSync(p);
    if (s.isDirectory()) walk(p, acc);
    else if (SUPPORTED_EXT.has(extname(p).toLowerCase())) acc.push(p);
  }
  return acc;
}

async function run() {
  const files = walk(PUBLIC_IMAGES_DIR);
  console.log(`Found ${files.length} images under public/images/`);

  let created = 0;
  let skipped = 0;
  let failed = 0;

  for (const absPath of files) {
    const relPath = relative(join(process.cwd(), "public"), absPath).replace(/\\/g, "/");
    const urlPath = `/${relPath}`;
    const fullUrl = `${BASE_URL}${urlPath}`;

    const existing = await db.select().from(images).where(eq(images.url, urlPath)).limit(1);
    if (existing.length) {
      skipped++;
      continue;
    }

    let width: number | null = null;
    let height: number | null = null;
    let sizeBytes = 0;
    try {
      const buf = readFileSync(absPath);
      const size = imageSize(buf);
      width = size.width ?? null;
      height = size.height ?? null;
      sizeBytes = buf.byteLength;
    } catch (e) {
      console.warn(`probe failed for ${urlPath}`, e);
    }

    let ai = null;
    try {
      ai = await autoTagImage(fullUrl);
      console.log(`✓ tagged ${urlPath}`);
    } catch (e) {
      console.warn(`AI tag failed for ${urlPath}:`, e);
      failed++;
    }

    const filename = urlPath.split("/").pop() || urlPath;

    await db.insert(images).values({
      source: "public",
      url: urlPath,
      pathname: relPath,
      filename,
      width,
      height,
      sizeBytes,
      mimeType: mimeFromExt(extname(absPath)),
      description: ai?.description ?? null,
      material: ai?.material ?? null,
      subjects: ai?.subjects ?? [],
      shotType: ai?.shotType ?? null,
      who: ai?.who ?? [],
      setting: ai?.setting ?? null,
      descriptionAutoGenerated: !!ai?.description,
      tagsAutoGenerated: !!ai,
      reviewed: false,
    });

    created++;
    await sleep(400);
  }

  console.log(`\nDone. Created: ${created}, skipped: ${skipped}, AI-failed: ${failed}`);
  process.exit(0);
}

function mimeFromExt(ext: string) {
  const m: Record<string, string> = {
    ".jpg": "image/jpeg",
    ".jpeg": "image/jpeg",
    ".png": "image/png",
    ".webp": "image/webp",
    ".avif": "image/avif",
  };
  return m[ext.toLowerCase()] ?? "application/octet-stream";
}

function sleep(ms: number) {
  return new Promise((r) => setTimeout(r, ms));
}

run().catch((e) => {
  console.error(e);
  process.exit(1);
});

## 10. Gotchas and caveats

1. **OpenRouter image format.** Use `type: "image_url"` with `image_url: { url: ... }` — OpenAI-compatible format, NOT Anthropic's `type: "image"`. Already handled in `image-ai.ts`.

2. **Vercel Blob URLs aren't on jespermakes.com.** Expected. If someone asks, it's intentional — a custom domain on the Blob store is a later concern. Existing `/images/...` URLs remain unchanged.

3. **AI failures are non-fatal.** Upload still succeeds, the image enters the library with empty tags and `reviewed: false`. Jesper tags it manually. Never throw from AI failure into the user-facing upload flow.

4. **Serverless timeout on upload.** Set `maxDuration = 60` on the upload route. Typical upload + tag round trip is 5-10s; 60s gives headroom for large images on slow networks.

5. **Rate limiting.** None in v1. If the admin endpoint gets hammered we'll add `@upstash/ratelimit` the same way we plan to for the Title Lab AI routes.

6. **Image size lib.** `image-size` is tiny and sync. Don't bring in `sharp` — it's a binary native dep and breaks Vercel builds without specific configuration.

7. **The controlled vocabulary is a TypeScript file, not DB-backed.** Deliberate simplification. If Jesper asks to add a tag, edit `src/data/image-tag-vocabulary.ts`, commit, redeploy. Takes ~90 seconds. Admin-UI vocabulary editing is a v1.1 concern.

8. **ESLint apostrophes.** Standard site-wide gotcha — any `'` in JSX text must be `&apos;`. Check before `next build`.

9. **`sql\`array&&\`` for array overlap.** Drizzle doesn't have first-class `&&` support, which is why `findImages` and the list endpoint use raw SQL for overlap queries. Tested with Postgres; if schema changes move to JSONB later this syntax needs updating.

10. **Migration re-runs are safe.** The script checks `url` uniqueness and skips existing rows. Run it once, run it again if AI fails on some images — it'll only retry the missing ones.

## 13. `tool-knowledge-image-library.md` (goes in project folder, not repo)

```markdown
# The Image Library — Knowledge Document

Reference doc for `/admin/images` and the `findImages()` helper. Paste into
future Claude chats when building anything that needs to pick site images.
Sibling to `tool-knowledge-box-joint-jig.md`.

## What it is

Admin page at `/admin/images` plus a `findImages()` helper function. Every
image on the site has a row in the `images` table with description, material,
subjects, shot type, who's in it, setting, and custom tags. The UI lets Jesper
upload, review, and tag. The helper lets Floki (and any future Claude) query.

Solves the original problem: Floki was picking wrong images (timber framing
when asked for plywood) because he was listing public/images/ and guessing
from filenames. Now he has to query the library.

## The non-negotiable rule

Before ANY Claude writes `heroImage`, `featuredImage`, or a `/images/*` path
anywhere in the codebase, that Claude must call `findImages()` first. No
exceptions. If no result fits, stop and request an upload. Do not guess.

## The API

```typescript
import { findImages } from "@/lib/image-library";

// By vocabulary field
await findImages({ material: "plywood" });
await findImages({ subjects: ["jig", "box-joint"] });
await findImages({ shotType: "hero", setting: "workshop" });

// Free-text search across description + filename + tags
await findImages({ query: "workshop overview warm light" });

// Combined
await findImages({
  material: "plywood",
  shotType: "detail",
  limit: 5,
});
```

Returns an array sorted by match score (descending). `result[0]` is the best.
Each item is an `Image` row plus `matchScore: number`.

## The vocabulary

Controlled, lives in `src/data/image-tag-vocabulary.ts`. Single source of truth.

- **material** (one): plywood, pallet-wood, oak, walnut, pine, birch, mixed-wood
- **subjects** (many): jig, planter, box-joint, cabinet, shelf, tool-closeup, workshop, finished-piece
- **shotType** (one): hero, detail, process, b-roll, portrait
- **who** (many, optional): jesper, wife, hands-only
- **setting** (one, optional): workshop, outdoors, studio
- **customTags** (many, freeform): whatever else

To add a vocabulary value, edit the file and redeploy. ~90 seconds.

## Storage

- Pre-existing images: `public/images/...`, URL is `/images/...` (relative).
- New uploads: Vercel Blob, URL is full `https://...blob.vercel-storage.com/...`.

Both are queryable identically. `findImages()` returns a `url` field for each
— use it as-is in img src. Don't worry about which source.

## Tagging quality

AI tags on upload and on "regenerate." Jesper reviews and corrects in the UI.
The `reviewed` flag flips true the first time Jesper saves from the editor.
Unreviewed images are shown with an amber dot in the grid.

If a search turns up only unreviewed matches, consider flagging that to
Jesper — tags might not be accurate yet.

## Admin-only

Gated by `ADMIN_EMAIL`. No public gallery. No per-user image libraries.
The tool is for Jesper and for Floki, not for visitors.

## What's NOT here in v1

- Vocabulary editing from the admin UI (edit the TS file instead)
- Bulk tag edits across multiple images
- "Find similar" suggestions
- Public gallery
- Per-image usage tracking (we have a static `blog-posts.ts` / `tools.ts`
  scan for "used in", but no live view counts)
- Image optimization / resizing — still using raw uploads. `next/image`
  migration is a separate project.
